# Choose Your Prototype Output

You've completed the design package. Now it's time to bring it to life.

## Your Options

### 1. Figma Design
Best for: Designers who work in Figma and want to iterate visually.
What you get: A complete Figma file with your design system, components, and key screens.
Requirements: Figma account, Figma MCP server configured in Claude Code.

### 2. Design Specification
Best for: Handing off to a development team.
What you get: A comprehensive document with design tokens, component specs, layout grids, interaction patterns.
Requirements: Nothing extra.

### 3. Storybook Design System
Best for: Teams using component-driven development.
What you get: A Storybook project with your design tokens, base components, and composed views.
Requirements: Node.js installed.

### 4. Next.js Prototype
Best for: Demos, user testing, investor pitches, or starting development.
What you get: A working, clickable application with your design applied.
Requirements: Node.js installed.

## How to Choose

| I want to... | Choose |
|---|---|
| Explore visually and iterate in a design tool | Figma Design |
| Hand off to a developer or engineering team | Design Specification |
| Build a reusable component library | Storybook |
| Demo to users, investors, or stakeholders | Next.js Prototype |
| Start real development | Next.js Prototype |

Tell Claude: "I want to create a [Figma design / design spec / Storybook / Next.js prototype]"

Claude will read the appropriate guide and walk you through it.

## You Only Need One

You are not required to complete all four formats. Choose one and go deep. The other formats are available if your needs change later.
